# Data Directory Structure

## data/kg/

Knowledge Graph and embedding files.

| File | Description | Generated By |
|------|-------------|-------------|
| `triples.csv` | KG triples (134K: co_liked, acted_by, has_genre, directed_by, released_in_decade) | Phase 2.1: `build_kg.py` |
| `entity2id.csv` | Entity-to-ID mapping (14K entities) | Phase 2.1 |
| `kg_graph.pkl` | NetworkX graph object | Phase 2.1 |
| **`rotate_entity_emb.npy`** | **RotatE entity embeddings (128-dim, balanced sampling, 300 epochs)** | **Phase 2.2: `rotate.py`** |
| `rotate_relation_emb.npy` | RotatE relation phase embeddings | Phase 2.2 |
| `rotate_relation2id.json` | Relation-to-ID mapping (5 relations) | Phase 2.2 |
| `kg_features_*_recall.csv` | Hand-crafted KG features (IDF-weighted actor/genre/director sharing, co_liked, decade) | Phase 2.6 |
| `kg_emb_features_*_recall.csv` | RotatE embedding features (mean/min distance, mean/max cosine) | Phase 2.7 |
| `kg_paths_for_recommendations.json` | KG connection paths for top-10 recommendations (for Phase 2 RAG) | `export_phase1_for_rag.py` |

### Note on embedding model

The project uses **RotatE** (not TransE) as the KG embedding model. RotatE models relations as rotations in complex space, better handling symmetric relations (co_liked) and N-to-N relations (has_genre, acted_by). See `docs/EXPERIMENT_LOG.md` Experiments 3-5 for the comparison study.

`kg/transe.py` is kept for baseline comparison. The main pipeline (`run_all.py`) uses `kg/rotate.py`.
